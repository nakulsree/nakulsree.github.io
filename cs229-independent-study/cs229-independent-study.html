I

Why I Did This

An Industrial Engineering degree teaches optimization, statistics, and operations research — but not the theory of machine learning. By the time I was building models on real projects, I kept hitting a wall: I could run algorithms, but I couldn't reason about why they worked, when they would fail, or how to diagnose problems at a mathematical level.

Stanford's CS229 was the answer. The course materials — lecture videos, lecture notes, and problem sets — are publicly available. The problem sets require implementing core algorithms from scratch, not calling library functions. That distinction matters: writing a Naive Bayes classifier by hand forces you to understand Laplace smoothing and log-likelihood in a way that sklearn.naive_bayes never will.

I worked through this in evenings and weekends over roughly a year. I wrote handwritten notes through each lecture. I implemented every algorithm between the START CODE HERE and END CODE HERE markers myself. The scaffolding was provided; the mathematical substance was mine to derive.

What CS229 covers
CS229 is Stanford's graduate-level ML course. Unlike introductory ML courses that treat models as black boxes, CS229 derives everything from probabilistic foundations — maximum likelihood, exponential families, EM, kernel theory, MDPs. It is the course that trained a generation of Silicon Valley ML engineers.
What makes it different
The assignments require implementing algorithms numerically from their mathematical definitions — not using scikit-learn, not using PyTorch. Each problem set problem traces from theory to code: derive the update rule on paper, then implement it in Python.
Connection to my projects
This theoretical grounding directly informs decisions in my applied work: understanding EM's convergence properties shaped how I thought about the routing ensemble in the outage forecasting project; understanding generative vs. discriminative tradeoffs informed model selection in the SpaceX classification pipeline.

01
Logistic Regression via Newton's Method
SupervisedClassification

Gradient descent converges, but slowly — each step is the same size regardless of the curvature of the loss surface. Newton's method uses the Hessian to take a step proportional to the inverse curvature, achieving quadratic convergence near the optimum. The parameter update is θ := θ − H⁻¹∇θℓ(θ), where H = Xᵀ diag(h(xᵢ)(1−h(xᵢ))) X.

θ := θ − H⁻¹ ∇θ ℓ(θ)  ·  H = Xᵀ diag(hi(1−hi)) X
# Hessian: H = X^T diag(h*(1-h)) X
H = X.T @ np.diag(h * (1 - h)) @ X
# Newton step
theta -= np.linalg.solve(H, grad)
The convergence difference vs. gradient descent on a poorly-conditioned dataset is dramatic — Newton's method converges in ~5–10 iterations where gradient descent needs thousands. This experience made "conditioning" feel concrete rather than abstract.
02
Gaussian Discriminant Analysis (GDA)
GenerativeClassification

GDA models the class-conditional distributions P(x|y) as Gaussians with separate means but shared covariance, then applies Bayes' rule. Unlike logistic regression (discriminative), GDA makes stronger assumptions about the data distribution — and pays off with closed-form MLE solutions when those assumptions hold.

μj = Σ 𝟙[y⁽ⁱ⁾=j] x⁽ⁱ⁾ / Σ 𝟙[y⁽ⁱ⁾=j]  ·  Σ = (1/m) Σ (x⁽ⁱ⁾−μy⁽ⁱ⁾)(x⁽ⁱ⁾−μy⁽ⁱ⁾)ᵀ
mu_0 = X[y==0].mean(axis=0)
mu_1 = X[y==1].mean(axis=0)
phi = (y==1).mean()
diff = X - np.where(y[:,None]==1, mu_1, mu_0)
Sigma = (diff.T @ diff) / m
The key insight: GDA's decision boundary is linear in x when the two classes share a covariance matrix — the same family as logistic regression — but logistic regression is more robust when the Gaussian assumption is violated. Knowing when each is appropriate requires understanding both.
03
Positive-Unlabeled (PU) Learning
Label NoiseCorrection

A rarely-taught but practically important problem: what if you only have positive labels — you know which examples are class 1, but you have no confirmed negatives, only unlabeled examples? Standard logistic regression on partial labels will systematically underestimate P(t=1|x). The correction: estimate α = P(y=1|t=1) on a validation set, then P(t=1|x) = h(x)/α.

α = P(y=1|t=1) = E[h(x) | y=1]  ·  P̂(t=1|x) = h(x) / α
# Estimate alpha on validation set
alpha = predict_val[y_val==1].mean()
# Corrected probability
p_corrected = clf.predict(x_test) / alpha
This problem structure appears constantly in practice: fraud detection (you know confirmed fraud, not confirmed non-fraud), medical diagnosis (confirmed positives, but negatives may be undetected positives). Understanding the correction factor is what separates a practitioner who can handle real data from one who assumes clean labels.
04
Poisson Regression — Generalized Linear Models
GLMCount Data

The exponential family framework unifies linear regression, logistic regression, Poisson regression, and others under a single derivation. For count targets, Poisson regression models y|x ~ Poisson(λ) with λ = exp(θᵀx), ensuring non-negative predictions. The gradient ascent update falls directly from the log-likelihood.

∂ℓ/∂θⱼ = Σᵢ (y⁽ⁱ⁾ − exp(θᵀx⁽ⁱ⁾)) x⁽ⁱ⁾ⱼ
# Gradient ascent (maximizing log-likelihood)
h = np.exp(x @ self.theta)
grad = x.T @ (y - h) / m
self.theta += self.step_size * grad
This directly informed later project work. The outage forecasting problem — predicting integer counts with severe right skew — is exactly the distributional regime where Poisson or negative binomial regression is appropriate. Understanding GLM theory made the choice of loss functions principled rather than heuristic.
05
EM Algorithm for Gaussian Mixture Models
UnsupervisedSemi-Supervised

EM alternates between assigning soft cluster memberships (E-step) and updating cluster parameters to maximize expected log-likelihood (M-step). I implemented both the unsupervised version (K=4 Gaussians, random initialization) and the semi-supervised extension that incorporates a weighted cross-entropy term for labeled examples. The semi-supervised version converges faster and is more robust to initialization.

E-step: wj⁽ⁱ⁾ = φⱼ · N(x⁽ⁱ⁾; μⱼ, Σⱼ) / Σₖ φₖ · N(x⁽ⁱ⁾; μₖ, Σₖ)
# E-step: compute soft responsibilities
w[:,j] = phi[j] * multivariate_normal.pdf(x, mu[j], sigma[j])
w /= w.sum(axis=1, keepdims=True)
# M-step: weighted MLE
mu[j] = (w[:,j,None] * x).sum(0) / w[:,j].sum()
The semi-supervised EM result (convergence in far fewer iterations, better cluster assignments) was a direct empirical demonstration of why labeled data is valuable even in small quantities. The unsupervised version's sensitivity to initialization made "local optima in non-convex optimization" viscerally clear.
06
Locally Weighted Regression
Non-ParametricRegression

Standard linear regression fits a single global model. Locally weighted regression re-fits θ for every query point x, giving higher weight to training examples close to x. The bandwidth parameter τ controls the effective neighborhood size — a direct knob on the bias-variance tradeoff. I compared six values of τ and identified τ=0.05 as optimal on the validation set.

w⁽ⁱ⁾ = exp(−||x⁽ⁱ⁾−x||² / 2τ²)  ·  θ = (XᵀWX)⁻¹XᵀWy
# Compute kernel weights for query point x_i
w = np.exp(-np.sum((X - x_i)**2, axis=1) / (2 * self.tau**2))
W = np.diag(w)
theta = np.linalg.solve(X.T @ W @ X, X.T @ W @ y)
07
Kernel Perceptron
KernelsOnline Learning

The kernel trick allows any algorithm that operates only through inner products to be "kernelized" — learning in an implicitly high-dimensional feature space without ever computing that representation. The perceptron in its dual form stores the training examples and makes predictions via f(x) = Σ αᵢ K(xᵢ, x), updating αᵢ only on misclassification.

f(x) = sign(Σᵢ αᵢ y⁽ⁱ⁾ K(x⁽ⁱ⁾, x))  ·  update: αᵢ += 1 if misclassified
# Predict using kernel dot products
score = sum(b * kernel(x_s, x_i)
       for b, x_s in zip(state[0], state[1]))
# Update: accumulate support vectors on mistake
state[0].append(y); state[1].append(x_i)
08
Independent Component Analysis (ICA)
UnsupervisedSource Separation

Given a set of mixed audio signals — each microphone recording a superposition of five simultaneous sound sources — ICA recovers the original independent sources by learning an unmixing matrix W. The Laplace distribution is used as the source prior (heavy-tailed, unlike Gaussian), and W is updated via gradient ascent on the log-likelihood. Applied to an actual five-channel audio mixing problem.

W := W + α ( (Wᵀ)⁻¹ − sign(Wx) xᵀ )
# Gradient ascent update on W
# Laplace prior: g'(s) = sign(s)
updated_W = W + lr * (
  np.linalg.inv(W).T
  - (np.sign(W @ x))[:,None] @ x[:,None].T
)
The key insight about ICA vs. PCA: PCA finds directions of maximum variance (decorrelation); ICA finds directions of statistical independence (much stronger condition). The Laplace prior is chosen specifically because Gaussian sources are fundamentally unidentifiable by ICA — a subtle theoretical constraint with practical consequences.
09
Naïve Bayes Spam Classifier + SVM
NLPClassification

A full text classification pipeline: tokenize SMS messages, build a vocabulary of words appearing in ≥5 messages, construct a word-count feature matrix, and fit a multinomial Naïve Bayes model with Laplace smoothing. Predictions via log-likelihood comparison to avoid floating-point underflow. Also implemented SVM radius hyperparameter search via validation set accuracy.

φk|y=j = (Σ 𝟙[xᵢ=k ∧ yᵢ=j] + 1) / (Σ 𝟙[yᵢ=j] + |V|)
# Log-likelihood prediction (avoids underflow)
score_1 = np.log(phi_y) + X @ np.log(phi_one)
score_0 = np.log(1-phi_y) + X @ np.log(phi_zero)
return (score_1 > score_0).astype(int)
Results: Naïve Bayes achieved 97.8% test accuracy. Top 5 spam-indicative words: claim, won, prize, tone, urgent! SVM with optimal radius 0.1 achieved 96.9%. Naïve Bayes outperformed SVM on this dataset — consistent with the theoretical result that generative classifiers can outperform discriminative ones with limited data.
10
Reinforcement Learning — CartPole via Value Iteration
MDPValue Iteration

Model-based RL on the classic CartPole problem: balance a pole on a cart by applying left or right force. The continuous state space (x, ẋ, θ, θ̇) is discretized into 163 states. The agent learns an MDP — estimated transition probabilities P̂(s'|s,a) and rewards R̂(s) — from experience, then applies value iteration to compute the optimal policy. The loop: act → observe → update model → re-run value iteration whenever the pole falls.

V(s) := R(s) + γ · maxa Σs' P(s'|s,a) V(s')  ·  γ = 0.995
# Value iteration (Bellman update)
V_new = R + gamma * np.max(
  V @ P[:,:,:], # shape: (states,actions)
  axis=2).ravel()
converged = np.max(np.abs(V_new - V)) < tol
Convergence criterion: 20 consecutive value function computations all converging in a single iteration. This is the point where the agent's model of the world has stabilized — further failures yield negligible updates. The log of time-steps-to-failure increases monotonically as the agent improves, making learning directly visible.
III

What This Built

The value of this study isn't in the 10 implementations — it's in the cross-cutting theoretical frameworks that make all of them coherent.

The MLE–MAP–Bayesian Hierarchy

Every supervised learning algorithm in CS229 can be situated within a single probabilistic framework. Maximum likelihood estimation (logistic regression, Poisson regression), maximum a posteriori with regularization (ridge, LASSO), and full Bayesian inference all emerge from different assumptions about priors. Seeing this unified picture changes how you diagnose overfitting — it's not a "trick," it's a prior.

Discriminative vs. Generative Models

Logistic regression models P(y|x) directly. GDA models P(x|y) and P(y), then applies Bayes' rule. GDA uses more assumptions and pays off when they hold; logistic regression is more robust. This tradeoff — modeling efficiency vs. robustness to misspecification — recurs constantly in applied work and informs how I evaluate model choices.

The Exponential Family Unification

Linear regression, logistic regression, and Poisson regression are all instances of the same GLM framework: choose an exponential family distribution for y|x, use the canonical response function as your prediction, and gradient ascent on log-likelihood always has the same form: θⱼ := θⱼ + α(y⁽ⁱ⁾ − h(x⁽ⁱ⁾))xⱼ⁽ⁱ⁾. One derivation, many algorithms.

The EM Framework and Local Optima

EM guarantees convergence to a local maximum of the log-likelihood — not the global maximum. This is not a limitation to apologize for; it's a feature of the landscape of latent variable models. Understanding that EM is coordinate ascent on a lower bound (Jensen's inequality applied to log-likelihood) makes the convergence proof intuitive, not magical.

Kernel Methods and the Dual Representation

Any algorithm expressible through inner products can be kernelized. This includes SVMs (via the dual Lagrangian), the perceptron (dual form), and kernel regression. The representer theorem says that the optimal solution lives in the span of the training data — a deep result that connects all of these. Kernel choice is a modeling decision about the geometry of the problem.

MDPs and the Bellman Equations

Value iteration is a contraction mapping. The Bellman optimality operator is guaranteed to converge to the unique fixed point V*. This is not just an algorithmic fact — it's why reinforcement learning with known dynamics is tractable. The CartPole implementation made the MDP framework concrete: states, actions, transition probabilities, rewards, discount factor — each is a meaningful modeling decision.
