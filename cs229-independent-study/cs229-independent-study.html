I

Why I Did This

My IE coursework covers optimization and statistics, but it did not give me the ML theory I wanted. I could run models, but I was not always confident about why they worked or how to debug them when they failed.

Stanford's CS229 was the cleanest way to close that gap. The assignments are all about implementing algorithms from scratch, so there is no hiding behind libraries. Writing Naive Bayes by hand makes Laplace smoothing and log-likelihood feel real, not abstract.

I worked through it over about a year on nights and weekends. I took handwritten notes, then implemented every algorithm between the START CODE HERE and END CODE HERE markers myself. Stanford provided the scaffolding; the math and code were on me.

◆
A note on framing: The problem structure and scaffolding belong to Stanford. What I contributed is the implementation of the mathematical core of each algorithm — and more importantly, the understanding of why each line of code is justified by the underlying theory. This page is an honest account of that.
What this ML study covers
This study follows Stanford's CS229 syllabus: probabilistic foundations, maximum likelihood, exponential families, EM, kernel methods, and MDPs. The emphasis is on deriving and implementing the math, not just running libraries.
What makes it different
The assignments require implementing algorithms numerically from their mathematical definitions — not using scikit-learn, not using PyTorch. Each problem set problem traces from theory to code: derive the update rule on paper, then implement it in Python.
Connection to my projects
This theoretical grounding directly informs decisions in my applied work: understanding EM's convergence properties shaped how I thought about the routing ensemble in the outage forecasting project; understanding generative vs. discriminative tradeoffs informed model selection in the SpaceX classification pipeline.

III

Theoretical Foundations

The value is in the cross-cutting frameworks that unify all these algorithms.

Probabilistic Unification

MLE, MAP, and Bayesian inference sit on a spectrum of prior assumptions. Linear regression, logistic regression, and Poisson regression are all GLMs — one derivation, many algorithms.

Discriminative vs. Generative

When to model P(y|x) vs. P(x|y) and P(y). Each choice is a tradeoff between modeling efficiency and robustness to violated assumptions.

Kernel Methods & Kernelization

Any algorithm using only inner products can be kernelized. The representer theorem connects this to the geometry of the problem.

EM & Local Optima

EM is coordinate ascent on a lower bound. Local optima are not a flaw but a feature of latent variable models.

MDPs & Value Iteration

Value iteration is a contraction mapping guaranteed to converge. State, action, and discount factor are all meaningful modeling choices.
