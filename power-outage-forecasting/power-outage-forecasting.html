01 / Overview

Executive Summary

This project addresses a deceptively hard forecasting problem: predicting hourly power outage counts across all 83 Michigan counties, 48 hours ahead. The difficulty is structural — roughly 70% of county-hours record zero outages, yet the rare extreme events (a single storm can spike a county to 23,346 outages in one hour) are precisely what utilities need to anticipate.

Rather than fitting one global model that inevitably under-weights the tail, the approach partitions the prediction problem into four data-driven magnitude regimes and trains specialized models for each. A LightGBM meta-learner then refines the routing using out-of-fold disagreement signals across all base models, correcting boundary misclassifications at inference time.

The resulting framework achieves an 89% reduction in Winkler score over a global residual quantile baseline and a 43% reduction in normal-case RMSE from the meta-learner alone — with prediction intervals calibrated to be narrow during quiet periods and appropriately wide during extreme events.

02 / Problem

Problem Statement

Standard regression on power outage counts fails because the target distribution violates the assumptions of a single global model:

  • Severe zero-inflation: 70.47% of county-hours record zero outages. A mean-squared-error objective is dominated by these structural zeros, predicting near-zero for most samples.
  • Heavy tails: Variance exceeds the mean by more than 10×. The June 26, 2023 storm drove CAPE to 14.6× its seasonal background, producing county-averaged hourly counts of 1,040 — a regime completely unlike the quiet baseline.
  • Event-driven, non-periodic structure: Unlike electricity demand, outages have no day-of-week or hour-of-day cycle. The distribution is punctuated by rare, physically catastrophic events rather than smooth seasonality.
  • Competing objectives: Minimizing normal-period error (s1) and capturing extreme-event severity (s2/s3) pull the gradient in opposite directions for any single loss function.

05 / Methodology

Approach & Architecture

Stage 1
Multi-Class Router

A LightGBM classifier assigns each county-hour to one of four magnitude regimes, identified via k-means clustering on the non-zero outage distribution (k=3 selected by silhouette score, boundaries validated by Wasserstein distance):

  • Class 0: Zero outages (structural zeros)
  • Class 1: Low activity [1, 10)
  • Class 2: Moderate [10, P80 ≈ 299]
  • Class 3: Extreme tail (>P80)

Class 0 predicts exactly zero; Class 1 predicts the within-class training mean. Diversity is reserved for Classes 2 and 3 where tail underprediction was the primary failure mode.

Stage 2
Regime-Specific Regressors

Three complementary regressors trained independently for Classes 2 and 3:

  • LightGBM + Huber Loss: Downweights extreme residuals while preserving gradient signal for moderate values. Transition parameter δ tuned per regime.
  • LightGBM + Log-Log Transform: Models ỹ = log(1 + log(1 + y)), yielding near-Gaussian residuals within Class 3 and stable gradients at extreme counts.
  • MLP (128→64, ReLU, MSE): Introduces smooth non-linear feature interactions beyond tree-split approximations. Adam optimizer with L2 decay.

Final class prediction: arithmetic mean of successfully trained regressors.

ỹ = log(1 + log(1 + y))
ŷ = exp(exp(ŷ˜) − 1) − 1
Stage 3
Bayesian Hyperparameter Optimization

Model parameters tuned via Tree-structured Parzen Estimator (TPE) in Optuna with successive halving pruning. Key parameters:

  • Tree depth, learning rate, regularization strength
  • Huber transition parameter δ (optimized independently for Classes 2 and 3)

Objective: F1 score on the most recent validation fold, representing conditions closest to the test window. Parameters optimized independently per regime, reflecting the fundamentally different error landscapes of moderate vs. extreme events.

Stage 4
Meta-Learner

A second LightGBM classifier trained on a 10-dimensional out-of-fold meta-feature vector:

z = [p₀, p₁, p₂, p₃, ŷ¹L, ŷ²L, ŷ³L, ŷ¹H, ŷ²H, ŷ³H]

Exposes: router soft-probability uncertainty, regressor agreement or disagreement across models, and patterns in which model performs best per meteorological configuration. Trained entirely on out-of-fold predictions to prevent overfitting to in-sample behavior.

Validation Strategy

5-fold expanding window: each fold adds ~2 weeks of data and holds out the following block. Fold boundaries mirror the day-of-week composition of the test window. Final test predictions average all five fold models.

Prediction Interval Construction

A magnitude-scaled adaptive approach replaces the inadequate fixed-width global residual quantile baseline:

L = max(0, 0.75 · ŷ)
U = max(L + 3, 1.35 · ŷ)

Scale parameters (0.75, 1.35) and minimum width (3) jointly tuned on out-of-fold residuals from all five folds, targeting ~95% empirical coverage across 175,296 validation samples while minimizing expected Winkler score under the competition's penalty structure.

Key Insights

01
The meta-learner's 43% RMSE gain comes from exploiting regressor disagreement — when the three base regressors disagree, that disagreement is a signal about routing uncertainty at the Class 2/3 boundary, which the meta-learner learns to correct.
02
The Winkler improvement (89%) is primarily architectural, not parametric — the magnitude-scaled interval directly addresses the heteroscedasticity that defeats any fixed-width approach.
03
Precision (0.893) substantially exceeds recall (0.571) — predicted non-zero events are highly reliable, but ~43% of actual outage county-hours are missed, predominantly low-signal events with absent outage history and weak meteorological forcing.
04
The Lifted Index's LOW→HIGH importance shift (+0.028) is the largest of any feature, consistent across both Huber and LogLog frameworks — confirming it as the primary thermodynamic driver of severe outage magnitude in Michigan.
07 / Reflections

What I Learned

🧩

Mixture-of-Experts Over Single Complex Models

When relationships between features and targets are highly complex, a single over-parameterized model (even transformers and LSTMs) can fail to overfit the training data in meaningful ways. Decomposing into specialized sub-models — each optimized for a distinct regime — consistently outperformed any single architecture attempted.

⚖️

Loss Function Design for Skewed Data

With severely right-skewed targets, standard MSE is not robust to outliers — yet those outliers are often the most important predictions. The solution was a deliberate ensemble of loss functions: Huber for moderate events, log-log transformation for extreme counts, and standard MSE in the MLP for capturing spikes. Each loss function encodes a different tradeoff between robustness and tail sensitivity.

📏

Loss Normalization Across Components

When combining heterogeneous losses (e.g., F1 from a classifier alongside count-scale regression loss), normalization is critical. Unnormalized losses on different scales cause one component to dominate the training signal, which can silently degrade performance on the component whose loss appears smaller in magnitude.

🚫

Strict Data Leakage Prevention in Time Series

In a real forecasting problem, present-period variables cannot be used to predict the present target — in practice, those variables won't exist at inference time. All meteorological features were shifted one hour backward relative to the target, and the class boundary threshold was recomputed within each training fold only. This discipline is non-negotiable for valid out-of-sample evaluation.

🔁

Expanding Window vs. Standard Cross-Validation

Standard k-fold shuffles temporal order, causing future data to "leak" into training folds for time series. Expanding window validation preserves chronological order — each fold adds more data and holds out the next block — making it the only valid cross-validation scheme for this problem structure.

🔎

Early Hyperparameter Tuning ROI Assessment

Extensive Bayesian optimization (Optuna, TPE) can yield diminishing returns — in this project, early iterations sometimes produced <5% improvement across hours of compute. The lesson: run a short pilot of hyperparameter search first, assess marginal gain per trial, and only invest deep if the trajectory is promising. Architecture choices often matter more than fine-grained parameter tuning.

🧪

Ablation as a Debugging Strategy

Building incrementally — starting from the base model and adding one component at a time — allowed pinpointing exactly which architectural decision (e.g., the meta-learner, the log-log transform, the magnitude-scaled intervals) drove which performance change. This ablation discipline made the results interpretable and the paper's claims defensible.

🌡️

Physical Interpretability Validates Architecture

The zero-overlap between router and severity features is not just a performance result — it is a physical validation. The fact that detection relies on temporal autocorrelation while severity relies on atmospheric thermodynamics confirms that the two-stage architecture is capturing genuinely different sub-problems, not just partitioning noise.
